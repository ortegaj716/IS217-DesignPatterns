Memento + Facade

What are Mementos and Facades?

If you know about transactions in InnoDB type databases, then mementos should be familiar territory. Mementos are containers for data sets. Whenever you need to save an object's attributes, you would put them in a memento, and then put the memento in an array. As you keep making mementos, you would start to accumulate a history for that object. If for some reason you need to rollback to a previous state, you could restore your object's data set to that of a previous memento in the array.

Facades are function that provide easy high level access to very complicated objects. They obscure the inner workings to present a very easy and workable interface to programmers (Somewhat similar to the factory design pattern; it hides the complicated object creation by presenting an easy interface).
